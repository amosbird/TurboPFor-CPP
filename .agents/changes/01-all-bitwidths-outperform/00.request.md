# Change Request: Make All Bit Widths Outperform TurboPFor C

## Context

We have a C++ reimplementation of TurboPFor's `p4d1dec128v64` (delta-1 P4 decode for 128-element blocks of 64-bit integers). The implementation uses a hybrid approach: periodic-unroll for bit widths where the period P > 2, and fully-unrolled for P ≤ 2.

## Current State

Average performance is **+3.2%** faster than TurboPFor C across b=1..32 (SIMD path, no exceptions). However, several bit widths are still slower or barely even:

| BitWidth | Decode Diff | Status |
|----------|-------------|--------|
| 2 | -0.2% | ❌ Negative |
| 4 | -0.9% | ❌ Negative |
| 30 | -1.2% | ❌ Negative |
| 31 | -0.4% | ❌ Negative |
| 32 | +0.1% | ⚠️ Barely positive |

## Goal

Make **every single bit width** (b=1..32) consistently outperform TurboPFor C in D1 decode benchmarks, while maintaining correctness (binary_compat_test, vbyte64_test must pass).

## Constraints

- No `__uint128_t` (ClickHouse portability)
- Must compile with clang 18.1.3
- Must pass all existing tests
- Cannot regress any currently-positive bit widths
